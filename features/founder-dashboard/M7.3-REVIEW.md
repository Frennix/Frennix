## Delivered

### Secure staff onboarding (no per-user SQL)

| Flow | How it works |
|------|----------------|
| **First owner** | One-time `founder-bootstrap-init` Edge Function sets token hash → owner claims at `/founder/bootstrap` |
| **Ongoing staff** | Owner/Founder creates invite in **Admin** → share link → invitee accepts at `/staff/join` |
| **Revoke** | Admin UI → revoke membership (audit logged) |
| **Audit trail** | Every invite create/accept/cancel/revoke/bootstrap claim logged to `founder_audit_log` |

### Expanded roles & permissions

8 roles with table-driven capabilities (`staff_role_capabilities`):

| Role | Access level |
|------|----------------|
| **Owner** | Full control including owner assignment |
| **Founder** | Full ops except owner/founder invites |
| **Admin** | Roadmap, releases, ambassadors, all dashboards |
| **Moderator** | Moderation + community activity |
| **Support** | Support tools + community activity |
| **Ambassador Manager** | Ambassadors + community analytics |
| **Content Manager** | Content moderation + analytics |
| **Analyst** | Read-only all analytics dashboards |

Frontend + backend checks:
- `can_access_founder_dashboard()` RPC gates Settings link and shell
- `has_staff_capability()` RPC gates every dashboard RPC and nav item
- Settings link hidden from normal users (`canAccessDashboard === false`)

### Community Health (`/founder/community`)

Tracks all requested metrics:
- DAU, WAU, MAU, new signups, retention (D1/D7/D30)
- Workout posts, stories, messages, events, challenges, matches
- Comments, reactions, referral growth, ambassador activity
- Date range filter, CSV/JSON export, 7-day trend table
- Auto-refreshes via `refresh_founder_metrics_daily()` on load

### Platform Health (`/founder/platform`)

Tracks all requested subsystems:
- App errors, crash reports (placeholder probe), API latency
- Supabase health, Realtime messaging, database health
- Storage usage (placeholder), notification delivery (placeholder)
- Deployment status from `app_releases`
- Overall status banner, CSV/JSON export, 30s auto-refresh

### Admin UI (`/founder/admin`)

- Create staff invite (email + role) → copy invite link
- List active staff (search, revoke)
- Pending invites (cancel)
- Audit trail (search, export)

### First-time owner setup (one-time infra, not per-user SQL)

```bash
# 1. Set a long random secret
supabase secrets set FOUNDER_BOOTSTRAP_SECRET=your-long-random-secret-here

# 2. Deploy and invoke bootstrap init (once)
supabase functions deploy founder-bootstrap-init
curl -X POST "https://<project-ref>.supabase.co/functions/v1/founder-bootstrap-init" \
  -H "Authorization: Bearer <SUPABASE_SERVICE_ROLE_KEY>"

# 3. Sign in to Frennix, visit /founder/bootstrap, enter the secret
# 4. Invite additional staff from /founder/admin
```

### Existing founder from M7.1 manual SQL

If you already ran manual SQL to grant `founder` role, bootstrap is skipped (`has_owner = true`). Use **Admin → Invite Staff** for additional team members.
